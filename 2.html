第一步：准备环境（只需做一次）

// 初始化本地项目仓库 git init // 关联远程版本库 git remote add origin https://github.com/xiaofneg666/lian.git // 克隆远程代码下来本地 git clone xxxx

第二步：日常开发流程（循环使用）

// 1. 当前分支状态查看（改动总览） git status // 2. 修改的代码细节展示 git diff // 3. 把所有改动放进暂存区（Git 2.x后 . 和 -A 作用相同，包含修改、新增和删除） git add . git add -A // 4. 提交暂存区文件到本地仓库 git commit -m "这次我写了什么功能" // 5. 拉取最新远程代码并快速合并 git pull // 6. 提交本地数据到远程分支 (例如 master) git push origin master

第三步：分支管理与切换

// 查看本地对应远程的分支对应关系 git branch -vv // 查看本地和远程所有分支 git branch -a // 显示所有本地分支 git branch // 切换到已有的分支 git checkout xxxx git switch xxxx // 创建新分支但不切换 git branch dev // 以当前分支为基础新建并切换到xxx分支 git checkout -b xxx // 将本地分支与远程xxx分支进行关联 git branch --set-upstream-to=origin/xxx // 删除本地分支（当前分支不能为待删除分支） git branch -d dev // 删除远程分支 (两种写法) git push origin --delete dev git push origin :dev

第四步：合并与衍合 (Merge & Rebase)

// 合并指定分支到当前分支（产生合并节点） git merge xxxx // 衍合指定分支到当前分支（线性历史） git rebase xxxx // 如果 rebase 遇到冲突，解决后： git add . git rebase --continue

第五步：回滚、撤销与强制推送

HEAD 说明： HEAD^ 表示上一个版本，HEAD^^ 表示上上个版本。
// 查看历史纪录（精简版） git log --oneline // 修改的部分代码清理掉不修改了（丢弃工作区修改） git checkout . // 撤销所有已经 add 的文件： git reset HEAD . // 撤销某个文件或文件夹的 add 状态： git reset HEAD filename // 撤销 commit 之后返回成暂存区 add 状态（保留代码） git reset --soft HEAD^ // 撤销 commit 连带代码改动直接消失（不保留代码） git reset --hard HEAD^ // 强行推送到远程版本（慎用） git push -f -u origin master

第六步：代码贮藏 (Stash)

// 贮藏修改（代码没写完又需要切分支时使用） git stash save "备注信息" // 查看所有贮藏记录 git stash list // 释放最近一次贮藏并恢复到当前分支 git stash pop // 恢复指定的某个贮藏 git stash pop stash@{$num}

第七步：远程库信息与标签 (Remote & Tags)

// 查看远程版本库详细信息 git remote -v git remote show origin // 从远程库获取代码但不合并 git fetch origin // 上传所有本地标签 (Tags) git push --tags // 删除远程标签 git push origin :refs/tags/tag-name // 修剪掉已经删除远程分支的本地缓存 git remote prune origin
