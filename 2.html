第一步：准备环境（只需做一次）

本地项目初始化与远程关联：

// 1. 初始化本地仓库
git init

// 2. 关联远程 GitHub 仓库
git remote add origin https://github.com/xiaofneg666/lian.git

// 3. 克隆远程代码（如果是从零开始开发他人项目）
git clone <url>

第二步：日常开发（不断循环）

1. 状态查看与对比

// 当前分支状态（改动总览）
git status

// 查看具体的代码修改细节
git diff

2. 提交到暂存区

把所有改动放进暂存区（别忘了那个点！）：

// 监控工作区状态树，提交所有变化（含修改、新增、删除）
git add . （注意空格和点）
git add -A （与点作用相同）

3. 提交到本地仓库

// 正式存入本地仓库并写好备注
git commit -m "这次我写了什么功能"

4. 推送到远程

// 把本地进度同步到 GitHub 对应分支
git push origin master

// 拉取最新远程代码（防止冲突）
git pull

第三步：分支管理

// 查看本地对应远程的分支对应关系
git branch -vv

// 查看本地和远程所有分支
git branch -a

// 创建新分支
git branch dev

// 切换分支
git switch 分支名 （或 git checkout 分支名）

// 新建并直接切换到该分支
git checkout -b xxx

// 合并 xxx 分支到当前分支
git merge xxx

// 删除本地分支（不能在当前分支删自己）
git branch -d dev

// 删除远程分支
git push origin --delete dev

第四步：怎么回滚与撤销

可以先看看之前的历史纪录：git log --oneline

// 1. 撤销工作区修改（清理掉不想改的代码）
git checkout .

// 2. 文件退出暂存区，但保留修改
git reset HEAD .

// 3. 撤销 commit，回到已 add 状态
git reset --soft HEAD^

// 4. 彻底回退到某个版本，不保留代码改动
git reset --hard <commit_id>

// 5. 强推到远程（谨慎使用）
git push -f

第五步：代码贮藏 (Stash)

// 写错分支了？先存起来
git stash save "备注"

// 查看存了哪些
git stash list

// 切换分支后取出代码
git stash pop

第六步：其他常用配置

// 查看远程版本库信息
git remote -v

// 查看项目所有 Git 配置
git config --list

// 修剪已经失效的远程分支记录
git remote prune origin
