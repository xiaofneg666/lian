分支合并与衍合 (Merge vs Rebase)

核心区别：
• Merge 会产生一个新的合并节点，保留完整的历史分支图谱。
• Rebase 会重新定义起点，让提交历史变成一条直线，看起来更整洁，但会改变提交 ID。
// 合并指定分支到当前分支（保留分支结构）
git merge <branch>

// 衍合指定分支到当前分支（使历史线性化）
git rebase <branch>

// 如果 rebase 遇到冲突，处理完冲突后继续：
git add .
git rebase --continue

远程操作 (Remote Operations)

// 查看远程版本库信息（显示简略/详细信息）
git remote -v

// 查看指定远程库（如 origin）的详细信息（含分支对应关系）
git remote show origin

// 添加新的远程版本库关联
git remote add <remote_name> <url>

// 从远程库获取最新代码到本地，但不自动合并
git fetch origin

// 下载代码并快速合并到当前分支 (相当于 fetch + merge)
git pull origin <branch>

// 上传本地代码及快速合并到远程分支
git push origin <branch>

标签与远程删除 (Tags & Delete)

// 删除远程分支（写法1：冒号法，较老旧）
git push origin :<branch-name>

// 删除远程分支（写法2：现代写法，语义更清晰 - 推荐）
git push origin --delete <branch-name>

// 删除远程标签 (Tag)
git push origin :refs/tags/<tag-name>

// 将本地所有标签一次性上传到远程
git push --tags

// 将单个指定标签上传到远程
git push origin <tag-name>
小贴士： 在执行 git push -f (强制推送) 之前，务必确保你清楚这会覆盖远程代码，通常只在个人分支或 Rebase 之后使用。
